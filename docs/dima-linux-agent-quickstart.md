# Инструкция для Димы: Linux Agent

Архив для установки: `DC_via_Cudy-linux-prod.zip`.

## Установка

1. Распаковать архив в отдельную папку.
2. Открыть терминал в этой папке.
3. Выполнить:

```bash
chmod +x *.sh
./one_click_install.sh
```

Если попросит пароль `sudo`, ввести пароль Linux-пользователя.

Установщик делает три вещи:

- восстанавливает прямой маршрут в интернет перед началом;
- при необходимости пытается скачать `sing-box` в `./runtime/sing-box`;
- запускает один smoke-тест и ставит systemd-сервис `cudy-managed-agent.service`.

Если скачивание бинарников запрещено или интернет нестабилен, надо заранее
положить `sing-box` в `runtime/sing-box` и запустить:

```bash
AUTO_INSTALL_SINGBOX=0 ./one_click_install.sh
```

## Проверка

После установки выполнить:

```bash
./status.sh
```

Нормальные признаки:

- control tunnel отвечает;
- service `cudy-managed-agent.service` активен;
- в managed transports есть только реально нужные транспорты;
- в конце лога есть строки `routes applied` или `probe jobs processed`;
- интернет не пропадает при включенной AmneziaVPN/агенте.

## Откат

Если интернет сломался или нужно полностью убрать агент:

```bash
sudo ./uninstall_systemd.sh
```

Если нужно только вернуть прямой маршрут без удаления сервиса:

```bash
sudo ./restore_direct.sh
```

## Что отправить для диагностики

Если что-то не работает, достаточно прислать полный вывод:

```bash
./status.sh
```

Этот скрипт ничего не меняет. Он показывает маршруты, DNS, firewall-подсказки,
активные VPN/TUN-интерфейсы, состояние control tunnel и последние строки лога.
